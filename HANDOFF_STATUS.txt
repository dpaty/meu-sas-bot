🎯 GARANTIA: BOT PARA QUANDO HUMANO ASSUME

Flow de Bloqueio (3 Camadas):

1️⃣  ConversationController.avaliarConversa()
    └─ Verifica: Está silenciado? → NÃO RESPONDE

2️⃣  InstanceManager.handleIncomingMessage()
    └─ Re-valida antes de processar IA
    └─ Re-valida ANTES de enviar resposta

3️⃣  Lock de processamento
    └─ Evita respostas duplicadas

═══════════════════════════════════════════════════════════

✨ COMO USAR (3 Formas)

1️⃣  VIA CURL (Tester/Backend)
────────────────────────────
# Assumir atendimento
curl -X POST http://localhost:3000/api/handoff/silenciar/user123 \
  -H "Content-Type: application/json" \
  -d '{"chatId":"5521987654321@c.us"}'

# Liberar bot
curl -X POST http://localhost:3000/api/handoff/desilenciar/user123 \
  -H "Content-Type: application/json" \
  -d '{"chatId":"5521987654321@c.us"}'

2️⃣  VIA DASHBOARD (Frontend)
──────────────────────────
<script src="handoff-integrator.js"></script>

<script>
    const handoff = new HandoffIntegrator('seu_user_id');
    
    // Assumir
    await handoff.assumir('5521987654321@c.us');
    
    // Liberar
    await handoff.liberar('5521987654321@c.us');
    
    // Atualizar UI
    setInterval(() => handoff.atualizarUI(), 10000);
</script>

3️⃣  AUTOMÁTICO (Cliente Pede)
────────────────────────────
Cliente: "Quero falar com humano"
  ↓
Bot detecta palavra-chave
  ↓
Bot se silencia automaticamente
  ↓
Operador assume e responde

📊 ENDPOINTS DISPONÍVEIS

GET  /api/handoff/silenciados/:userId
     └─ Lista todos os chats em atendimento humano

POST /api/handoff/silenciar/:userId
     ├─ Body: {"chatId":"5521987654321@c.us"}
     └─ Ação: Operador assume, bot silencia

POST /api/handoff/desilenciar/:userId
     ├─ Body: {"chatId":"5521987654321@c.us"}
     └─ Ação: Libera bot, retoma automático

GET  /api/handoff/status/:userId/:chatId
     └─ Verifica: {estaSilenciado, status}

GET  /api/handoff/estado/:userId
     └─ Estado geral: {chatsEmAtendimento, processando, total}

🔒 VALIDAÇÕES REALIZADAS

✓ Instância WhatsApp ativa?
✓ Chat está silenciado?
✓ Já processando? (evita duplicação)
✓ Verificação ANTES de processar IA
✓ Verificação ANTES de enviar resposta
✓ Verificação ANTES de marcar processando
✓ Lock sempre liberado (try-finally)
